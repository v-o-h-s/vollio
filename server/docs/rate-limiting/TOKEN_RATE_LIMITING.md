# Token-Based Rate Limiting for AI Calls

This document describes the token-based rate limiting system for controlling AI usage per user.

## Overview

The system tracks AI token consumption using **weighted tokens** and enforces:

- **Daily limits**: Prevents burning entire monthly quota in one day
- **Monthly limits**: Overall cap on AI usage
- **Burst capacity**: Maximum tokens allowed per single request (for large documents)

## Token Weighting

Tokens are weighted to account for cost differences:

| Token Type                 | Weight | Rationale                       |
| -------------------------- | ------ | ------------------------------- |
| Prompt tokens (input)      | 1x     | User-provided input             |
| Completion tokens (output) | 4x     | Generated by AI, more expensive |

**Formula**: `weighted_tokens = prompt_tokens * 1 + completion_tokens * 4`

## Default Limits

| Limit   | Value      | Description                        |
| ------- | ---------- | ---------------------------------- |
| Monthly | 10,000,000 | Total weighted tokens per month    |
| Daily   | 500,000    | Maximum weighted tokens per day    |
| Burst   | 500,000    | Maximum tokens in a single request |

These limits are customizable per-user via the `user_token_quotas` table.

## Architecture

```
Request → Token Rate Limiter Plugin → AI Service → OpenRouter
                    ↓                       ↓
                Redis (fast)            Token Usage
                    ↓                       ↓
              Supabase (persistent)   Record Usage
```

### Components

1. **`TokenRateLimitingService`**: Core service for quota checks and usage recording
2. **`TokenQuotaRepository`**: Supabase persistence layer
3. **`tokenRateLimiterPlugin`**: Fastify preHandler hook for route enforcement
4. **`token_quota.lua`**: Lua script for atomic Redis operations

## Configuration

### Route Configuration

Enable token rate limiting on AI routes:

```typescript
fastify.post(
  "/generate",
  {
    config: {
      rateLimit: { cost: 20, category: "ai" },
      tokenRateLimit: true, // Enable token quota check
    },
  },
  handler
);
```

### Environment Variables

| Variable     | Default     | Description       |
| ------------ | ----------- | ----------------- |
| `REDIS_HOST` | `127.0.0.1` | Redis server host |
| `REDIS_PORT` | `6379`      | Redis server port |

## API Response Headers

Every AI endpoint returns quota information:

| Header                      | Description                        |
| --------------------------- | ---------------------------------- |
| `X-Token-Daily-Remaining`   | Remaining daily token quota        |
| `X-Token-Monthly-Remaining` | Remaining monthly token quota      |
| `X-Token-Daily-Reset`       | Unix timestamp of next daily reset |

## Rate Limit Exceeded Response

When quota is exceeded, the server returns:

**Status Code**: `429 Too Many Requests`

```json
{
  "success": false,
  "message": "Token Quota Exceeded",
  "data": null,
  "error": {
    "type": "token_quota_exceeded",
    "reason": "daily_limit",
    "remaining": {
      "daily": 0,
      "monthly": 5000000
    },
    "retryAfter": 3600,
    "message": "You have reached your daily AI usage limit. Your quota will reset at midnight UTC."
  }
}
```

### Reason Codes

| Reason          | Description                                    |
| --------------- | ---------------------------------------------- |
| `daily_limit`   | Daily quota exceeded, resets at midnight UTC   |
| `monthly_limit` | Monthly quota exceeded, resets on 1st of month |
| `burst_limit`   | Single request exceeds burst capacity          |

## Database Schema

### `user_token_quotas`

Stores per-user limits and usage:

| Column               | Type      | Description                        |
| -------------------- | --------- | ---------------------------------- |
| `user_id`            | uuid      | Primary key, references auth.users |
| `monthly_limit`      | bigint    | Monthly token limit                |
| `daily_limit`        | bigint    | Daily token limit                  |
| `burst_capacity`     | bigint    | Max tokens per request             |
| `monthly_used`       | bigint    | Current monthly usage              |
| `daily_used`         | bigint    | Current daily usage                |
| `last_daily_reset`   | timestamp | Last daily reset time              |
| `last_monthly_reset` | timestamp | Last monthly reset time            |

### `token_usage_logs`

Audit log for analytics:

| Column              | Type      | Description               |
| ------------------- | --------- | ------------------------- |
| `id`                | uuid      | Primary key               |
| `user_id`           | uuid      | User reference            |
| `prompt_tokens`     | integer   | Raw prompt tokens         |
| `completion_tokens` | integer   | Raw completion tokens     |
| `weighted_tokens`   | bigint    | Calculated weighted total |
| `model`             | text      | AI model used             |
| `endpoint`          | text      | API endpoint called       |
| `created_at`        | timestamp | When the request occurred |

## Redis Keys

| Key Pattern                    | Description                 |
| ------------------------------ | --------------------------- |
| `token:quota:{userId}:daily`   | Daily usage counter         |
| `token:quota:{userId}:monthly` | Monthly usage counter       |
| `token:quota:{userId}:limits`  | Cached limits from database |

## Reset Schedule

- **Daily**: Midnight UTC (00:00 UTC)
- **Monthly**: First day of each month at midnight UTC

Reset is handled automatically:

1. Lua script checks reset timestamps
2. If quota needs reset, counters are zeroed
3. Supabase functions can be scheduled via cron for backup

## Usage in Use Cases

Use cases should record token usage after AI calls:

```typescript
const result = await this.generativeAiService.generateText(prompt);

await this.tokenRateLimitingService.recordUsage(userId, {
  promptTokens: result.usage.promptTokens,
  completionTokens: result.usage.completionTokens,
  model: result.model,
  endpoint: "my-endpoint",
});
```
