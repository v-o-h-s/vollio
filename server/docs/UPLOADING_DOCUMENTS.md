# Document Uploading Process

Vollio uses a two-step secure upload process to allow clients to upload files directly to Supabase Storage, reducing server load and improving performance.

## The Upload Flow

1. **Step 1: Request Signed Upload URL**
   - The client sends the document name (and optional folder context) to the server.
   - The server generates a unique, time-limited, and path-restricted **Signed Upload URL**.
   - The server returns the `storageUrl` (for the upload) and the `storagePath` (for step 2).

2. **Step 2: Direct Upload to Storage**
   - The client performs a `PUT` request directly to the returned `storageUrl`.
   - The binary data is sent directly to Supabase Storage.
   - The client ensures the upload is successful (HTTP 200).

3. **Step 3: Finalize Document Metadata**
   - The client sends a request to the server to "finish" the upload.
   - This request includes the `storagePath` from Step 1, along with metadata like file size and name.
   - The server validates that the `storagePath` belongs to the authenticated user and creates the database record.

## API Reference

### 1. Generate Upload URL
Used to get a secure destination for the document.

- **URL**: `/api/v1/documents/upload-url`
- **Method**: `POST`
- **Body**:
  ```json
  {
    "name": "manual_biology_final.pdf"
  }
  ```
- **Response**:
  ```json
  {
    "success": true,
    "data": {
      "storageUrl": "https://supabase.../upload/...",
      "storagePath": "user-uuid/1736041234_manual_biology_final.pdf"
    }
  }
  ```

### 2. Direct Storage PUT
Performed by the client.

- **URL**: `storageUrl` (from Step 1)
- **Method**: `PUT`
- **Headers**:
  - `Content-Type`: `application/pdf` (or original mime type)
- **Body**: `File` (binary buffer)

### 3. Finish Upload
Finalizes the metadata and creates the document in the DB.

- **URL**: `/api/v1/documents/finish-upload`
- **Method**: `POST`
- **Body**:
  ```json
  {
    "name": "manual_biology_final.pdf",
    "size": 1048576,
    "storagePath": "user-uuid/1736041234_manual_biology_final.pdf",
    "folderId": "optional-folder-uuid"
  }
  ```
- **Response**:
  ```json
  {
    "success": true,
    "message": "Document created successfully",
    "data": {
      "id": "new-document-uuid"
    }
  }
  ```

## Security Measurements

- **User Isolation**: Every upload is prefixed with the user's UUID. The server verifies that the metadata creation request (`finish-upload`) matches the authenticated user's ID.
- **Path Verification**: The server rejects any `storagePath` that doesn't start with the user's ID or contains path traversal segments (`..`).
- **Signed URLs**: Upload URLs are only valid for 60 seconds (default) and only for the specific path generated by the server.
- **Cleanup**: If metadata creation fails, the server attempts to delete the orphan file from storage to prevent leaks.
