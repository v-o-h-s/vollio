# Error Handling and Problem Resolution

## Search and Research Guidelines

When encountering errors, bugs, or implementation challenges:

- Use the MCP Brave Search server to research solutions, documentation, and best practices
- Search for specific error messages, library documentation, and implementation patterns
- Look up Next.js 15, React 19, TypeScript, and Tailwind CSS specific solutions
- Research shadcn/ui component usage and Clerk authentication patterns

## Debugging Approach

1. **Identify the Problem**: Clearly define the error or issue
2. **Search for Solutions**: Use web search to find relevant documentation or similar issues
3. **Verify Compatibility**: Ensure solutions are compatible with the current tech stack versions
4. **Test Incrementally**: Apply fixes in small, testable increments
5. **Document Findings**: Note any important discoveries for future reference

## Common Search Patterns

- `"Next.js 15" + [specific error message]`
- `"React 19" + [component issue]`
- `"shadcn/ui" + [component name] + [problem]`
- `"Clerk authentication" + [auth issue]`
- `"TypeScript" + [type error]`
